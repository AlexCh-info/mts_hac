# Постановка задачи

МТС предоставила два источника геоданных о зданиях Санкт-Петербурга. Задача - объединить их в единый достоверный слой высотности зданий и сделать его пригодным для планирования сети базовых станций.

Конкретно это означало:

1. Проанализировать качество данных из обоих источников и разработать методы очистки
2. Придумать как сопоставить здания из разных источников - один дом там может быть представлен по-разному: разные полигоны, разные адреса, разные атрибуты
3. Определить наиболее достоверную высоту для каждого здания когда оба источника дают разные значения
4. Для зданий где высоты нет ни в одном источнике - оценить её по косвенным признакам с помощью ML
5. Провести валидацию результатов
6. Визуализировать итоговый слой на карте и сформулировать инсайты для планирования сети

# Как мы решили задачу

## Шаг 1 - Очистка и объединение источников (`data_cleaner_module`)

Два источника данных о зданиях СПб принципиально разные по структуре. Один содержит геометрию и теги объектов, другой - реестровые данные с адресами и высотами. Прежде чем что-то считать, нужно было привести их к единому виду и решить проблему сопоставления: одно физическое здание в разных источниках может быть представлено разными полигонами, с разной точностью геометрии и разными границами.

Для сопоставления использовали пространственный индекс STR-tree и индекс Жаккара (IoU). Здания с IoU >= 0.3 считали одним объектом. Для случаев many-to-many - граф связности на networkx, компоненты которого соответствуют одному физическому зданию. Результат - `merged_dataset.csv` с 155 439 зданиями и 69 признаками.

## Шаг 2 - ML-модель для предсказания высоты (`ml_module`)

16.6% зданий не имели высоты ни в одном источнике. Для них обучили LightGBM на зданиях с известной высотой. Основные признаки - геометрия здания, расстояние до центра города, средняя высота соседних зданий в радиусах 25/50/100/200м. Ключевой момент при обучении - Spatial Split, чтобы избежать утечки данных между географически близкими объектами. Итоговые метрики: MAE ~2.3м, RMSE ~3.1м, R² ~0.94. Результат - `filled_dataset.csv` с высотой для всех 155 439 зданий.

## Шаг 3 - Аналитика и инсайты (`analys_module`)

После того как данные готовы, провели полный анализ структуры застройки города. Проверили качество ML-предсказаний, разобрали высотность по типам зданий и районам, нашли блокировщики сигнала, сделали сеточный анализ 500x500м с тепловыми картами. Главная фича - кластеризация всех зданий на 3 зоны качества покрытия по индексу сложности, который считается через Mutual Information.

## Шаг 4 - Визуализация (`web_visualization_module`)

сети. Для этого:
