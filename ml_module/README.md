# Ход работы
## Изучение задачи
Первым делом стоит задача изучения ПО - это делается для того, чтобы понять суть задачи, данных и определить то, что мы хотим получить как конечный результат.
Как вы знаете из общего README и README части с обработкой датасета наша задача заключается в заполнении пропусков с помощью алгоритма МО. Однако, как профессионал
мы в первую очередь должны понять, реально ли нам нужен алгоритм МО для выполнения данной задачи. 

Тут мы приходим к особенности человеческого организма, а точнее к способности анализировать большие массивы информации. Дело в том, что человек способен 
обрабатывать до 1000 единиц информации (будь то строки, числа и т.д.) без каких-либо проблем, опечаток, пропусков. Но после этого порога человек начинает 
путаться, пропускать информацию и ошибаться. Поэтому ручная разметка сразу отпадает, как решение нашей задачи т.к. в нашем распоряжении примерно 170тыс. строк информации.

Переходим к следующему шагу. На этом шаге мы должны убедиться, что нет простого алгоритма (не МО), который помог бы нам решить эту задачу. Однако, посмотрев множество источников
информации по теме "Геоаналитика" (т.к. я впервые работаю с такой задачей), я пришел к выводу, что в нашем случае, я потеряю большое кол-во времени на поиск того, как реализовать этот простой алгоритм. Так что переходим к слю шагу.
