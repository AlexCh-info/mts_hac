## 1. SJoin
Функция sjoin из GeoPandas выполняет пространственное соединение двух гео-датафреймов на основе топологического предиката. В данном случае используется intersects — объекты считаются совпадающими, если их геометрии имеют хотя бы одну общую точку (пересекаются, касаются или один содержит другой).

Для каждой пары пересекающихся объектов вычисляется:

  -  IoU (Intersection over Union) — коэффициент качества совпадения.
  -  Расстояние между центроидами — дополнительная метрика близости.
  -  Пары с IoU ниже порога (MIN_IOU_THRESHOLD = 0.1) отбрасываются.

Преимущества sjoin:

  -  Использует оптимизированный пространственный индекс (R-tree) — поиск за O(log n) вместо O(n²).
  -  Нативная поддержка в GeoPandas — меньше кода, меньше багов.
  -  Автоматическая обработка сложных геометрий (MultiPolygon, отверстия).

## 1.1 cKDTree
```
coords_a = np.array([(g.centroid.x, g.centroid.y) for g in gdf_a.geometry])
tree_b = cKDTree(coords_b)
distances, indices = tree_b.query(coords_a, k=1)
```

Алгоритм поиска ближайшего соседа с использованием k-d дерева (k-dimensional tree) из библиотеки SciPy.

Как работает:
  -  Для каждого объекта вычисляется центроид (геометрический центр).
  -  Координаты центроидов источника Б преобразуются в k-d дерево — структуру данных, позволяющую выполнять поиск ближайшего соседа за логарифмическое время.
  -  Для каждого центроида источника А находится ближайший центроид в источнике Б.
  -  Если расстояние не превышает порог (MAX_DISTANCE_METERS = 50 м), пара считается потенциальным совпадением.

Почему используется как второй этап:
```
Сценарий: Здание из источника А не пересекается ни с одним объектом из Б,
          но находится рядом (погрешность оцифровки, разные системы координат).

Решение: Центроидный поиск "ловит" такие случаи, когда геометрии не пересекаются,
         но объекты явно представляют одно и то же здание.
```

## 1.2 IoU
Метрика качества пространственного совпадения двух полигонов.

Формула:
```
         Площадь пересечения
IoU = ─────────────────────────
      Площадь объединения
```
Интерпретация значений:
```
IoU = 1.0  → Идеальное совпадение (геометрии идентичны)
IoU = 0.5  → Половина площади общая, половина — разная
IoU = 0.1  → Минимальное допустимое перекрытие (порог из конфига)
IoU = 0.0  → Нет пересечения или ошибка вычисления
```

## 2. Разрешение many-to-one (many-to-many)
```
resolved = matches_df.loc[matches_df.groupby('id_a')['iou'].idxmax()].reset_index(drop=True)
```
Проблема: Одному объекту из источника А могут соответствовать несколько кандидатов из источника Б. И наоборот.

Пример:
```
Источник А: [Здание_123] (большой полигон)
Источник Б: [Гараж_456], [Дом_789], [Навес_012] (три маленьких полигона внутри Здание_123)

Результат sjoin: 3 совпадения с разными IoU
```

Стратегия разрешения: Для каждого id_a оставляем только одно соответствие — то, у которого максимальный IoU.

Почему максимальный IoU, а не минимальное расстояние:

  -  IoU учитывает и перекрытие, и размер — более комплексная метрика
  -  Минимальное расстояние может выбрать маленький объект рядом с большим, хотя они не связаны
  -  IoU естественным образом штрафует случаи "маленький внутри большого"

## 3. Слияние атрибутов: левое соединение с префиксами
```
merged = gdf_a.copy()  # За основа берётся источник А

# Добавляем информацию о сопоставлении
merged['matched_id_b'] = ...
merged['match_iou'] = ...
merged['match_type'] = ...

# Добавляем колонки из источника Б с префиксом "B_"
for col in gdf_b.columns:
    if col not in merged.columns and col != 'geometry':
        merged[f'B_{col}'] = merged['matched_id_b'].apply(
            lambda x: gdf_b_indexed.loc[x, col] if x in gdf_b_indexed.index else None
        )
```
Тип соединения: Левое соединение (left join) — сохраняются все записи из источника А, данные из Б добавляются только для найденных соответствий.

Почему именно левое соединение:

  -  Источник А считается основным (более полным или приоритетным)
  -  Здания без соответствия в Б не теряются — они просто остаются без дополнительных атрибутов
  -  Предсказуемое поведение: количество строк в результате = количество строк в источнике А

Обработка отсутствующих соответствий:
```
lambda x: gdf_b_indexed.loc[x, col] if x is not None and x in gdf_b_indexed.index else None
```

  -  Если для объекта из А не найдено соответствие в Б → значение остаётся None (NaN)
  -  Это корректно: отсутствие данных ≠ ошибка, модель должна уметь работать с пропусками

Логика агрегации высоты: правило MAX
```
def calculate_max_height(row):
    heights = []
    
    # 1. Прямое значение высоты из источника Б
    if pd.notna(row.get('B_height')) and row['B_height'] > 0:
        heights.append(row['B_height'])
    
    # 2. Расчётная высота: этажность × высота этажа
    if pd.notna(row.get('B_stairs')) and row['B_stairs'] > 0:
        floor_h = row.get('B_avg_floor_height', 3.0)
        heights.append(row['B_stairs'] * floor_h)
    
    # 3. Расчётная высота из источника А
    if pd.notna(row.get('gkh_floor_count_max')) and row['gkh_floor_count_max'] > 0:
        heights.append(row['gkh_floor_count_max'] * 3.0)
    
    return max(heights) if heights else None
```
Проблема: Высота здания может быть указана в разных источниках по-разному:

  -  Прямое измерение (лидар, кадастр).
  -  Расчёт через этажность и среднюю высоту этажа.
  -  Отсутствие данных в одном из источников.
Стратегия: Взять максимальное из всех доступных оценок.

### Главный метод: extract_all_features
Метод последовательно вызывает приватные методы для каждой категории признаков. Порядок выбран осознанно:
  -  Геометрические признаки — базовые, не зависят от других вычислений
  -  Пространственные признаки — требуют только геометрию и координаты
  -  Признаки окружения — требуют доступ ко всему датасету (gdf_all)
  -  Категориальные признаки — могут зависеть от предварительно извлечённых текстовых полей
  -  Статистические признаки — вычисляются относительно распределения других признаков, поэтому идут после них
  -  Адресные признаки — простые бинарные индикаторы, могут добавляться в конце
### Метод _add_geometry_features: признаки формы и размера
Детальное описание признаков:
  -  feat_log_area - логарифм площади
  -  feat_perimeter - периметр
  -  feat_compactness - компактность формы
  -  feat_elongation - вытянутость
  -  feat_vertices_count - количество вершин полигона
  -  feat_convex_hull_area - площадь выпуклой оболочки

Детальное описание признаков
